## 1. Google OAuth (소셜 로그인)

### 용도
사용자가 Google 계정으로 회원가입/로그인할 수 있도록 한다.

### 가입 및 설정 절차
1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 프로젝트 생성 또는 기존 프로젝트 선택
3. **API 및 서비스 > 사용자 인증 정보** 메뉴 진입
4. **OAuth 2.0 클라이언트 ID** 생성
   - 애플리케이션 유형: **웹 애플리케이션**
   - 승인된 리디렉션 URI 추가:
     - 로컬: `http://localhost:3000/oauth/callback`
     - 운영: `https://www.onetakestudio.duckdns.org/oauth/callback`
5. **OAuth 동의 화면** 설정
   - 범위(Scope): `email`, `profile`, `openid`

| Client ID | `51683382309-340jprf00qpfjggfv5jphuifk5jk1np0.apps.googleusercontent.com` |
| Client Secret | `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` |
| Redirect URI (운영) | `https://www.onetakestudio.duckdns.org/oauth/callback` |

### 사용하는 API 엔드포인트

## 2. Kakao OAuth (소셜 로그인)

### 용도
사용자가 카카오 계정으로 회원가입/로그인할 수 있도록 한다.

### 가입 및 설정 절차
1. [Kakao Developers](https://developers.kakao.com/) 접속 및 로그인
2. **내 애플리케이션 > 앱 추가** 메뉴에서 앱 생성
3. **앱 키** 메뉴에서 REST API 키 확인 (= Client ID)
4. **카카오 로그인** 활성화
   - **제품 설정 > 카카오 로그인** 메뉴에서 활성화 ON
5. **Redirect URI** 등록
   - 로컬: `http://localhost:3000/oauth/callback`
   - 운영: `https://www.onetakestudio.duckdns.org/oauth/callback`
6. **보안** 메뉴에서 Client Secret 생성 및 활성화
7. **동의항목** 설정
   - 필수: 닉네임, 프로필 사진
   - 선택: 카카오계정(이메일)

### 사용하는 API 엔드포인트

## 5. LiveKit (WebRTC 미디어 서버)

### 용도
WebRTC 기반 실시간 영상/음성 통신, 녹화, RTMP 멀티 플랫폼 송출을 처리한다.

### 구성 요소
1. **LiveKit Server**: WebRTC SFU (Selective Forwarding Unit)
2. **LiveKit Egress**: 녹화 (RoomComposite → MP4) 및 RTMP 스트림 송출
3. **LiveKit Ingress**: 외부 RTMP 스트림 수신

### 설치 방법

Docker Compose로 로컬 실행하거나, K8s에 배포한다. LiveKit은 오픈소스 프로젝트로 별도 가입이 필요 없다.

```bash
# Docker Compose 사용 (docker-compose.yml에 포함)
docker-compose up -d livekit livekit-egress livekit-ingress
```

### 키 생성 방법

LiveKit API Key/Secret은 자체적으로 설정한다. `livekit.yaml` 파일의 `keys` 섹션에 Key-Secret 쌍을 정의하면 된다.

```yaml
keys:
  devkey: devkey-secret-for-livekit-min-32-chars
```

운영 환경에서는 최소 32자 이상의 랜덤 문자열을 생성해서 사용해야 한다.

### 현재 설정 정보

| 항목 | 로컬 | 운영 |
|------|------|------|
| API Key | devkey | K8s Secret 관리 |
| API Secret | devkey-secret-for-livekit-min-32-chars | K8s Secret 관리 |
| HTTP 포트 | 7880 | 7880 |
| RTC TCP 포트 | 7881 | 7881 |
| UDP 범위 | 50200-50250 | 50000-60000 |
| RTMP 포트 | 1935 | 1935 |

### 설정 파일
- 로컬: `livekit.yaml`, `egress.yaml`, `ingress.yaml` (프로젝트 루트)
- 운영: `infra/k8s/livekit/server/configmap.yaml`, `infra/k8s/livekit/egress/configmap.yaml`
- 백엔드: `media-service/src/main/resources/application.yml` (livekit 섹션)

## 8. Vercel (프론트엔드 배포)

### 용도
Next.js 프론트엔드를 정적/서버리스 방식으로 배포한다.

### 가입 및 설정 절차
1. [Vercel](https://vercel.com/) 접속 및 GitHub/GitLab 계정으로 가입
2. **New Project**에서 GitLab 저장소 연결
3. **Framework Preset**: Next.js (자동 감지)
4. **Root Directory**: `frontend` 설정
5. **Environment Variables** 등록:
   - `NEXT_PUBLIC_GOOGLE_CLIENT_ID`
   - `NEXT_PUBLIC_KAKAO_CLIENT_ID`
   - `NEXT_PUBLIC_NAVER_CLIENT_ID`
   - `CORE_SERVICE_URL=https://i14c206.p.ssafy.io`
   - `MEDIA_SERVICE_URL=https://i14c206.p.ssafy.io`

### CI/CD 연동

GitLab CI에서 `frontend/` 변경 시 Vercel CLI로 자동 배포된다.

```yaml
# .gitlab-ci.yml 발췌
deploy-frontend:
  script:
    - npm install -g vercel
    - vercel deploy --prod --yes --token=$ONETAKESTUDIO_GITLAB_CI_TOKEN
```

## 9. AWS ECR (컨테이너 레지스트리)

### 용도
백엔드 서비스의 Docker 이미지를 저장하고, K8s에서 Pull하여 배포한다.

### 가입 및 설정 절차
1. [AWS Console](https://aws.amazon.com/) 접속 및 계정 생성
2. **ECR (Elastic Container Registry)** 서비스 진입
3. 리포지토리 생성:
   - `core-server`
   - `media-server`
4. AWS CLI 로그인:
   ```bash
   aws ecr get-login-password --region ap-northeast-2 | \
     docker login --username AWS --password-stdin $ECR_REGISTRY
   ```

### 설정 정보

| 항목 | 값 |
|------|-----|
| 리전 | ap-northeast-2 (서울) |
| 레지스트리 URL | GitLab CI Variables `$ECR_REGISTRY`에 저장 |
| Core 이미지 | `$ECR_REGISTRY/core-server:latest` |
| Media 이미지 | `$ECR_REGISTRY/media-server:latest` |

### 설정 파일
- `.gitlab-ci.yml` (docker 스테이지)

|------|-----|
| 도메인 | www.onetakestudio.duckdns.org |
