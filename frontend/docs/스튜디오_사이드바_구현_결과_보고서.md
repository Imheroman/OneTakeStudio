## 3. API 경로 및 동작

| 용도               | 경로                                           | 비고                                        |
| ------------------ | ---------------------------------------------- | ------------------------------------------- |
| 채팅 히스토리      | `GET /api/media/chat/{studioId}`               | V1 제외                                     |
| 채팅 전송          | `POST /api/media/chat`                         | body: `{ studioId, content, platform? }`    |
| 스튜디오 멤버 목록 | `GET /api/studios/{studioId}/members`          |                                             |
| 스튜디오 멤버 초대 | `POST /api/studios/{studioId}/members/invite`  | body: `{ email, role }`, 1명씩 여러 번 호출 |
| 녹화 시작          | `POST /api/recordings/start`                   | body: `{ studioId }`                        |
| 녹화 중지          | `POST /api/recordings/{studioId}/stop`         |                                             |
| 녹화 목록          | `GET /api/recordings/studio/{studioId}`        |                                             |
| 녹화 중인 항목     | `GET /api/recordings/studio/{studioId}/active` |                                             |
| 친구 목록          | `GET /api/favorites`                           | 래퍼 없음: `{ favorites, total, maxCount }` |
| 친구 검색          | `GET /api/favorites/search?q=`                 | 스튜디오 초대 모달에서 사용                 |

## 4. 동작 요약

- **사이드바**: 탭 클릭 시 해당 패널만 열림, 같은 탭 다시 클릭 시 패널 닫힘. 초기에는 아무 탭도 선택되지 않아 패널 미표시.
- **채팅 / 프라이빗채팅**: 동일 `StudioChatPanel` 사용. 프라이빗은 `filterPlatform="INTERNAL"`, 전송 시 `platform: "INTERNAL"`.
- **멤버**: 목록 조회 후 "사용자 초대" 클릭 시 `StudioInviteModal` 오픈. 친구 목록·검색·이메일 직접 입력 후 선택된 이메일마다 초대 API 1회 호출.
- **녹화**: 시작/중지/목록/진행 중 여부 연동. 중지 후 버튼이 즉시 "녹화 시작"으로 돌아가도록 `setActive(null)` 및 active API `data: null` 스키마 반영.
- **배너·에셋·노트·스타일**: 백엔드 미구현에 따라 목업 또는 로컬(sessionStorage)만 사용.
- **레이아웃**: 오른쪽 사이드바가 잘리지 않도록 루트에 `overflow-x-auto`, 메인 영역 `min-w-0 flex-1`로 조정.

## 5. 데이터 소스 및 송출 반영 (FAQ)

| 질문                                                            | 답변                                                                                                                                                                                                       |
| --------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **채팅은 실제 DB에서 가져오나요?**                              | **예.** 채팅은 `GET /api/media/chat/{studioId}`, `POST /api/media/chat` 로 **실제 백엔드(DB)** 와 연동됩니다. MSW 사용 시에만 목업 데이터가 나옵니다.                                                      |
| **배너·에셋·스타일은 목업인가요? 송출 영상에 반영 가능한가요?** | **배너·에셋·스타일은 목업/로컬 상태**입니다. 백엔드 API가 없어 **실제 플레이(송출) 영상에 반영하는 기능은 현재 구현 불가**입니다. 백엔드에서 해당 데이터 저장·송출 연동 API가 생기면 그때 연동 가능합니다. |
| **멤버 초대도 DB와 연동되나요?**                                | **예.** 멤버 초대는 `POST /api/studios/{studioId}/members/invite` 로 **실제 백엔드(DB)** 와 연동됩니다. MSW 사용 시에만 목업으로 동작합니다.                                                               |

## 7. 정리

- 채팅 REST는 **V1 제외** 경로(`/api/media/chat`)로 통일했고, 나머지(초대 다중 호출, 프라이빗 채팅 필터, 녹화/멤버 연동, MSW 보강)는 계획서 대로 반영했습니다.
- 스튜디오 페이지에서 `studioId`를 사이드바와 초대 모달에 전달하여, 채팅·녹화·멤버·초대가 동일 스튜디오 기준으로 동작합니다.
