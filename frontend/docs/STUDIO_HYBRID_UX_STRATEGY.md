# 스튜디오 하이브리드 UX 전략

**슬로건:** "준비는 OBS처럼, 송출은 StreamYard처럼"

---

## 1. 플로우 원칙: 씬 우선

- **씬을 먼저 만든 뒤**, 그 씬에 종속되는 소스를 추가하는 방식을 사용합니다.
- 씬이 선택되지 않은 상태에서는 **소스 추가 불가**. UI에서 "씬을 먼저 선택하세요" 안내.
- 프리뷰는 **선택된 씬**의 소스/레이아웃만 표시합니다.

---

## 2. 하이브리드 UX 두 축

### ① 샌드박스형 백스테이지 (Sandbox Backstage)

- 단순 리스트 대기가 아니라 **미니 캔버스(스테이징)** 를 두는 방식.
- **구현 방향**
  - 좌측 또는 하단에 **스테이징 캔버스** 영역을 둠.
  - 사용자가 이 영역에서 소스를 자유롭게 배치·크기 조절 (OBS 방식).
  - 준비가 끝나면 **송출(Go Live)** 버튼으로 메인 화면에 반영 (StreamYard 방식).
- **현재 상태:** 레이아웃 프리셋(full, split, grid) + 소스 ON/OFF로 프리뷰 구성. 스테이징 캔버스 상의 **드래그/리사이즈**는 추후 단계에서 도입 예정.

### ② 편집 모드 vs 라이브 모드

| 모드       | 설명 |
|------------|------|
| **편집 모드** (기본) | OBS처럼 소스 클릭·드래그·리사이즈 등 즉시 수정 가능. |
| **라이브 모드**     | 화면 **잠금(Lock)**. 소스 클릭해도 움직이지 않음. 레이아웃 프리셋·소스 ON/OFF만 활성화해 방송 사고 방지. |

- **구현 상태**
  - 헤더에 **잠금 / 편집** 토글로 모드 전환.
  - `isEditMode` 상태로 관리. 라이브 모드일 때는 향후 스테이징 영역의 드래그/리사이즈를 비활성화할 예정.

## 3. 구현 체크리스트

- [x] 씬 우선 플로우: 씬 없으면 소스 추가 불가, 프리뷰는 선택된 씬에 한함
- [x] 편집/라이브 모드 상태 및 헤더 잠금·편집 토글
- [x] `PreviewArea`에 `isEditMode` 전달 (향후 잠금 시 상호작용 비활성화용)
- [x] **레이아웃 저장** UI 제거(비우선화, 필요 시 추후 재도입)
- [x] 프리뷰 영역 **스테이징 / Live** 라벨 표시 (편집 모드 = 스테이징, 라이브 모드 = Live)
- [x] **Go Live** 클릭 시 자동 잠금(편집 → 라이브 모드 전환)
- [ ] 스테이징(미니 캔버스) 영역: 드래그·리사이즈 가능한 스테이징 캔버스 UI
- [ ] 라이브 모드 시 스테이징 영역 클릭/드래그 완전 비활성화
- [ ] Go Live 시 스테이징 → 메인 반영 로직 (StreamYard 스타일, 현재는 동일 캔버스)

---

## 4. 참고

- **OBS:** 로컬에서 소스 배치·스크린 구성에 강함.
- **StreamYard:** 브라우저 기반, 미리 준비한 레이아웃을 송출 시점에 적용하는 흐름.
- 본 전략은 **준비 단계는 OBS처럼 유연하게, 송출 단계는 StreamYard처럼 안전하게** 가져가기 위한 문서입니다.
- **레이아웃 저장:** 현 단계에서는 UI에서 제거(비우선). 필요 시 API·버튼 재도입 가능.
