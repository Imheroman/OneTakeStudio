# 스튜디오 미디어·캔버스·UX 점진 해결 계획

## 현재 이슈

1. **미디어 트랙 점유 문제**  
   같은 카메라/마이크를 PreviewArea(메인 캔버스)와 StagingSourceTile(미니 캔버스 타일)이 각각 `getUserMedia()`로 요청해, **동일 장치에 대해 스트림이 이중으로 열림**.  
   → 장치 사용 중 오류, LED 이중 점등, 리소스 낭비 가능.

2. **DOM 생명주기와 캔버스 드로잉 불일치**  
   React 마운트/언마운트·소스 배열 변경 시, 캔버스가 아직 등록되지 않은 소스를 그리려 하거나, 해제된 비디오 엘리먼트를 참조할 수 있음.  
   → 빈 프레임, 깜빡임, 드물게 크래시 가능.

3. **UI/UX 정리**  
   레퍼런스(PROGRAM/STAGING, 다크 테마, 인디고 액센트)와 맞추고, 라벨·간격·스타일을 기존 스튜디오와 일관되게 정리.

---

## Phase 1: 미디어 트랙 단일화 (우선)

**목표:** 소스당 `getUserMedia()` 1회만 호출, 스트림을 메인 프리뷰와 스테이징 타일이 공유.

| 단계 | 작업 | 상태 |
|------|------|------|
| 1.1 | `useSourceStreams(sources, options)` 훅 추가. 소스 ID별로 스트림 1개만 생성·보관·정리. | 완료 |
| 1.2 | StudioMain에서 `useSourceStreams(displaySources, …)` 호출 후, 반환 스트림을 PreviewArea·StagingArea에 전달. | 완료 |
| 1.3 | PreviewArea: `getSourceStream` prop 있을 때는 전달받은 스트림으로만 비디오 생성·등록, 내부에서 `getUserMedia` 호출하지 않음. | 완료 |
| 1.4 | StagingSourceTile: `stream` prop 있을 때는 해당 스트림을 `<video>`에만 연결, 타일 내부에서 `getUserMedia` 호출하지 않음. | 완료 |

**결과:** 같은 소스에 대해 스트림 1개만 유지 → 미디어 트랙 점유 이슈 해소.

---

## Phase 2: DOM/캔버스 동기화

**목표:** 캔버스 렌더 루프가 유효한 소스만 그리도록 하고, 해제된 엘리먼트를 참조하지 않도록 함.

| 단계 | 작업 | 상태 |
|------|------|------|
| 2.1 | useCanvasPreview 렌더 루프에서 sourceData.element 없으면 스킵(renderer에서 readyState 검사는 기존 유지). | 완료 |
| 2.2 | register 직후 한 프레임 대기 또는 “등록 완료” 플래그로, 등록 직후 빈 프레임 방지. | 예정 |
| 2.3 | unregister 시 해당 소스 즉시 맵에서 제거; 다음 render부터 해당 소스 스킵. | 기존 동작 유지 |

**결과:** 언마운트/순서 변경 시에도 캔버스가 깨진 참조를 쓰지 않음.

---

## Phase 3: UI/UX 정리

**목표:** PROGRAM/스테이징 구분 명확화, 기존 다크 테마·인디고와 통일.

| 단계 | 작업 | 상태 |
|------|------|------|
| 3.1 | 메인 영역 라벨: 편집 시 "프리뷰", 라이브 시 "Live". 빈 상태 문구 "프리뷰 영역" 등 통일. | 완료 |
| 3.2 | 스테이징 영역: "스테이징" 라벨·타일/버튼 스타일을 기존 패널과 톤 맞춤 (border-gray-700, bg-gray-800). | 완료 |
| 3.3 | "+ 소스 추가" 버튼 포커스 링, 타일 호버·드래그 시 border 일관. | 완료 |

---

## 구현 순서 요약

1. **Phase 1** 완료 → 미디어 트랙 단일화 적용.
2. **Phase 2**는 1 적용 후 실제 깜빡임/오류 재현 여부 보고 점진 적용.
3. **Phase 3**는 Phase 1과 병행 가능.
