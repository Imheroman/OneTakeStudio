# 스튜디오 프리뷰 로드맵 아이디어

레이어 순서(1=상단, 숫자 커질수록 뒤) 적용 이후 검토할 3가지 방향 정리.

---

## 1. 하이브리드 레이아웃: 스냅 가이드(Snap Guide) 시스템

**개념**: 완전 자유 드래그와 고정 템플릿의 중간 단계.

| 항목 | 내용 |
|------|------|
| **방식** | 기본은 스트림야드처럼 정해진 그리드 레이아웃. Shift 키 또는 '자유 모드' 활성화 시 크기·위치 자유 조절. |
| **타협** | 소스가 그리드/가이드 근처에 가면 **스냅(자석)** 으로 정렬. |
| **이점** | 초보자는 깔끔한 정렬 유지, "로고는 정확히 여기" 같은 세밀한 배치 니즈 충족. |

**구현 포인트**
- Konva `dragBoundFunc` 또는 드래그 종료 시 그리드/가이드 거리 계산 → 스냅 적용.
- 그리드 간격·가이드선 옵션(8px, 16px, 1/3 등) 설정 가능하게.
- Shift/자유 모드 토글 시 스냅 on/off.

## 2. 저장된 씬(Saved Scene) 기반의 정적 연산

**개념**: 매 순간 자유 배치 연산 대신, 확정된 상태만 렌더링해 CPU 부담 감소.

| 항목 | 내용 |
|------|------|
| **방식** | 편집 모드에서 소스 배치 후 '저장' → 좌표·크기를 고정 설정값(JSON)으로 저장. |
| **타협** | 실시간 송출 중에는 소스 직접 드래그 잠금, 미리 만든 **씬 간 전환(Transition)** 만 허용. |
| **이점** | 송출 중 불필요한 마우스/드래그 이벤트 감소, OBS급 화면 구성 자유도 유지. |

**구현 포인트**
- 씬별 `sourceTransforms` (또는 레이아웃 JSON)를 백엔드/로컬에 저장.
- Live 모드 진입 시 `draggable={false}`, Transformer 비활성화.
- 씬 전환 시 페이드/컷 등 전환 효과(선택).

---

## 3. 클라이언트 사이드 합성(Canvas) 중심 전략

**개념**: 서버 믹싱 대신 브라우저(Canvas)에서 합성 후 최종 결과만 서버로 송출.

| 항목 | 내용 |
|------|------|
| **방식** | Client-side Mixing: 서버에서 화면을 합치지 않고, 각 사용자 브라우저에서 Canvas로 합친 뒤 최종 스트림만 업로드. |
| **타협** | CPU 사용률이 임계치(예: 70%) 초과 시 복잡한 애니메이션 비활성화 또는 **적응형 프레임레이트/품질** 적용. |
| **이점** | 서버 비용 절감, 한국 고사양 PC 환경에서 네이티브에 가까운 선명한 화질. |

**구현 포인트**
- 현재 구조 유지: `getPreviewStreamRef` → Konva Layer 캔버스 `captureStream()` 로 이미 클라이언트 합성 후 스트림 생성.
- 추가: `PerformanceObserver` 또는 주기적 `performance.now()` 로 프레임 드롭/부하 감지 → 해상도/프레임레이트 다운 또는 효과 끄기.

## 우선순위·의존 관계 제안

| 순서 | 아이디어 | 이유 |
|------|----------|------|
| 1 | **저장된 씬 (2)** | 이미 씬/소스/트랜스폼 구조가 있음. "저장 → JSON" 및 Live 시 잠금만 붙이면 됨. 스냅·성능 정책의 기준이 됨. |
| 2 | **스냅 가이드 (1)** | UX 직결. 저장된 씬이 있어도 편집 시 스냅이 있으면 배치 품질이 올라감. |
| 3 | **적응형 성능 (3)** | Canvas 합성은 이미 하고 있으므로, "임계치 초과 시 조절"만 단계적으로 도입. |

## 구현 완료 (추천 순서대로 적용)

| 순서 | 아이디어 | 적용 내용 |
|------|----------|-----------|
| 1 | **저장된 씬 (2)** | 씬 전환 시 `layout.elements`에서 소스 + `transform`(x,y,width,height,zIndex) 복원. 편집 후 "씬 저장" 버튼으로 현재 배치를 API에 저장. Live 시 편집/저장 비활성화(씬 전환만 허용). |
| 2 | **스냅 가이드 (1)** | PreviewArea 드래그/리사이즈 종료 시 8px 그리드 스냅 + 스테이지 가장자리/중심 가이드(8px 이내 시 스냅) 적용. |
| 3 | **적응형 성능 (3)** | `useAdaptivePerformance` 훅: rAF로 프레임 시간 측정, 45ms 초과가 5프레임 연속이면 `onDegraded` 1회 호출(30초 쿨다운). 스튜디오에서 1080p일 때만 720p로 자동 하향. |
