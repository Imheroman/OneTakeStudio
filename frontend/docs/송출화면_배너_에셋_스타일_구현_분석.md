# 송출 화면 + 배너/에셋/스타일 구현 분석 보고서

> 현재 구현된 송출 화면·카메라/마이크/화면공유 구조를 정리하고, 배너·에셋·스타일을 송출 화면 위에 쌓는 구현 가능 여부와 필요 사항을 분석했습니다.

---

## 1. 현재 송출 화면·카메라/마이크/화면공유 구현 구조 (기술·구조 상세)

### 1.1 전체 흐름 요약

| 단계                    | 담당                                  | 기술                    | 설명                                                                                         |
| ----------------------- | ------------------------------------- | ----------------------- | -------------------------------------------------------------------------------------------- |
| 1. 소스 목록            | `useStudioMain`                       | React state             | 씬별 소스(video, audio, screen) 목록·온스테이지 여부·위치/크기(zIndex, x, y, width, height)  |
| 2. 미디어 스트림 획득   | `useSourceStreams`                    | **Web API**             | 카메라/마이크: `getUserMedia`, 화면공유: `getDisplayMedia` → `MediaStream` 반환              |
| 3. 송출 화면 그리기     | `PreviewArea`                         | **Konva (react-konva)** | Stage → Layer 위에 비디오/이미지 소스를 **Canvas**로 매 프레임 그리기                        |
| 4. 송출 스트림 내보내기 | `PreviewArea` + `getPreviewStreamRef` | **captureStream**       | Konva Layer의 HTMLCanvasElement에서 `captureStream(30)`으로 실시간 스트림 생성 (녹화/송출용) |

---

### 1.2 카메라·마이크·화면공유 연결 (useSourceStreams)

**파일**: `frontend/src/hooks/studio/useSourceStreams.ts`

- **역할**: 소스 타입별로 브라우저 **MediaStream**을 한 번씩 열어서 `Map<sourceId, MediaStream>`으로 보관하고, `getStream(sourceId)`로 넘겨줌.
- **카메라 (video)**
  - `navigator.mediaDevices.getUserMedia({ video: { deviceId, width, height, frameRate }, audio: isAudioEnabled })`
  - 지정된 디바이스(또는 기본 웹캠)에서 비디오+옵션으로 오디오 트랙까지 한 스트림으로 획득.
- **마이크만 (audio)**
  - `getUserMedia({ audio: deviceId ? { deviceId: { ideal: deviceId } } : true, video: false })`
  - 오디오 전용 소스일 때 사용.
- **화면공유 (screen)**
  - `navigator.mediaDevices.getDisplayMedia({ video: true, audio: isAudioEnabled })`
  - 브라우저가 창/탭/전체 화면 선택 UI를 띄우고, 선택한 화면의 스트림을 반환.
- **정리**:
  - 카메라/마이크 = **getUserMedia**
  - 화면공유 = **getDisplayMedia**
  - 둘 다 **MediaStream**을 반환하며, 이 스트림을 `<video>`에 `srcObject`로 넣어서 재생·표시함.

### 1.3 송출 화면 그리기 (PreviewArea + Konva)

**파일**: `frontend/src/widgets/studio/preview-area/ui/PreviewArea.tsx`

- **역할**: “송출 화면” = **Konva Stage** 하나. 그 안에 **Layer** 하나에서 비디오/이미지 소스를 배치하고, 매 프레임 갱신해서 **Canvas**에 그린다.
- **구성 요소**
  - **react-konva**: `Stage`, `Layer`, `Group`, `Image`, `Rect`, `Text`, `Transformer` 사용.
  - **Stage**: 고정 해상도(720p: 1280×720, 1080p: 1920×1080). 실제 DOM은 `<canvas>` 하나.
  - **Layer**: 그 위에 그리는 “한 장의 캔버스”. 여기에 배경 + 소스들 + (추가 예정) 배너/에셋을 그림.
- **비디오가 화면에 나오는 방식**
  - 각 비디오/화면공유 소스마다 `HTMLVideoElement`를 하나씩 만들고, `video.srcObject = stream`으로 위에서 받은 `MediaStream` 연결.
  - Konva의 `Image`는 “이미지 소스”를 받을 수 있는데, 여기에 **HTMLVideoElement**를 넘기면, 그 비디오의 현재 프레임이 이미지처럼 그려짐.
  - `VideoSourceNode` 안에서 **requestAnimationFrame**으로 매 프레임:
    - `img.image(video)` → 현재 비디오 프레임을 Konva Image에 반영
    - `layerRef.current.batchDraw()` → Layer를 다시 그리기
  - 즉, **Canvas 2D 위에 비디오 프레임을 매 프레임 이미지로 복사**하는 구조.
- **레이아웃**
  - `arrangeSourcesInLayout`(shared/lib/canvas/layout.ts): full / split / three-grid / four-grid 등에 따라 각 소스의 (x, y, width, height) 셀을 계산.
  - `sourceTransforms`: 드래그·리사이즈로 변경된 위치/크기/zIndex를 저장하고, 이 값이 우선 적용됨.
  - 각 소스는 **Group** 하나로 묶이고, 그 안에 `VideoSourceNode`(Image) 또는 `ImageSourceNode` 또는 오디오용 Rect가 들어감.
- **송출 스트림 내보내기 (녹화/라이브용)**
  - `getPreviewStreamRef.current`에 함수를 등록해 두고, 그 함수가 **Konva Layer의 내부 Canvas**를 가져와서  
    `canvas.captureStream(30)`을 호출.
  - 반환값은 **MediaStream** (캔버스 내용이 실시간으로 스트리밍됨).
  - 이 스트림을 **MediaRecorder**로 녹화하거나, WebRTC 등으로 송출에 사용하는 구조.

### 1.4 StudioMain에서의 연결

**파일**: `frontend/src/widgets/studio/studio-main/ui/StudioMain.tsx`, `features/studio/studio-main/model/useStudioMain.ts`

- **useStudioMain**:
  - 스튜디오/씬/소스 목록, `displaySources`, `sourceTransforms`, 레이아웃 타입, 해상도 등 관리.
  - `getPreviewStreamRef`를 useStudioMain에 넘기고, PreviewArea에 같은 ref를 넘겨서 “현재 송출 화면 Canvas의 스트림”을 밖에서 쓸 수 있게 함.
- **useSourceStreams**:
  - `sources`(useStudioMain에서 관리)를 받아서, video/audio/screen 타입에 맞게 `getUserMedia` / `getDisplayMedia` 호출.
  - 반환된 `getStream`을 PreviewArea에 `getSourceStream`으로 넘김.
- **PreviewArea**
  - `displaySources`, `getSourceStream`, `sourceTransforms`, `layout`, `resolution` 등을 받아서, 위에서 설명한 대로 Konva Stage 위에 비디오/이미지 소스를 그린다.

---

### 1.5 구조 요약 (쉽게 보기)

```
[소스 목록] useStudioMain (sources, displaySources, sourceTransforms)
      ↓
[미디어 스트림] useSourceStreams(sources) → getUserMedia / getDisplayMedia
      ↓
[송출 화면] PreviewArea
  - getSourceStream(sourceId) → MediaStream
  - 각 소스마다 <video> 생성, srcObject = stream
  - Konva Stage(Layer) 위에 Image(video)를 매 프레임 requestAnimationFrame으로 그리기
  - Layer의 Canvas → captureStream(30) → 녹화/송출용 MediaStream
```

- **Canvas / Konva**:
  - “송출 화면” 전체가 **Konva의 Stage = 하나의 Canvas**이고, 그 위에 비디오 프레임·이미지·사각형·텍스트 등을 **레이어**처럼 그리는 구조.
  - 배너·에셋·스타일도 **같은 Stage/Layer 위에 추가 노드(Rect, Text, Image, Group 등)**로 그리면, 기존 비디오 위에 “쌓아서” 보이게 할 수 있음.

## 2. 배너·에셋·스타일 송출 화면 반영 — 구현 가능 여부 및 필요 사항

### 2.1 결론 요약

| 기능   | 구현 가능 여부 | 방식                                                                                        |
| ------ | -------------- | ------------------------------------------------------------------------------------------- |
| 배너   | ✅ 가능        | Konva Layer에 하단 고정 영역 + Rect/Text 또는 티커(가로 스크롤 텍스트) 노드 추가            |
| 에셋   | ✅ 가능        | Konva Layer에 Image(이미지 URL) 또는 Rect+Text 등 오버레이 노드 추가                        |
| 스타일 | ✅ 가능        | 테마에 따라 소스 Group의 clip (원형/사각 등), 배너/에셋 색상, 배경색 등 Konva 속성으로 제어 |

현재 송출 화면이 **Konva Stage + Layer**로 그려지고, **captureStream**까지 연결되어 있으므로,  
배너/에셋/스타일을 **같은 Layer(또는 상단 Layer)에 노드로 추가**하면,  
화면에 보이는 그대로 녹화·송출에도 포함됩니다.

---

### 2.2 현재 상태 (배너·에셋·스타일과 송출 화면 연결)

- **StudioSidebar**
  - `StudioBannerPanel`, `StudioAssetPanel`, `StudioStylePanel`를 렌더링하지만,
  - **StudioMain에서 이 패널들로 “선택된 배너/에셋/스타일”을 넘기거나 받는 props가 없음.**
  - 즉, 패널 안에서의 선택/상태는 **로컬 state·sessionStorage**에만 있고, **PreviewArea와 연결되어 있지 않음.**
- **PreviewArea**
  - 배너/에셋/스타일을 그리는 **Konva 노드가 전혀 없음.**
  - 배경 Rect + (비디오/이미지 소스 Group들) + Transformer 만 있음.

그래서 “선택한 배너/에셋을 송출 화면에 쌓는다”는 동작을 하려면,  
**상태 끌어올리기 + PreviewArea에 전달 + Konva 노드 추가**가 필요합니다.

---

### 2.3 배너 기능 — 구현 방향

- **목표**: 선택한 배너를 **송출 화면 하단**에 표시 (사진 예시처럼 하단 바 + 텍스트, 필요 시 티커).
- **필요 사항**
  1. **상태 끌어올리기**
     - `StudioBannerPanel`의 “선택된 배너” (`BannerItem | null`)를 **StudioMain** 또는 전역/컨텍스트 state로 올리기.
     - 예: `activeBanner`, `setActiveBanner`를 StudioMain에서 관리하고, StudioSidebar(또는 Banner 패널)에 `onSelectBanner`로 전달.
  2. **PreviewArea에 전달**
     - `PreviewArea`에 props 예: `activeBanner: BannerItem | null` 전달.
  3. **Konva에 배너 노드 추가**
     - 기존 Layer **맨 위**(소스들보다 나중에 그리기)에:
       - 하단 고정 높이의 **Rect** (배경색)
       - **Text** (배너 텍스트)
       - `isTicker === true`이면 requestAnimationFrame으로 텍스트 x 위치를 이동시켜 티커처럼 스크롤.
     - 배너는 **listening: false**로 해서 드래그/선택 방해하지 않게 하는 것이 좋음.
- **데이터**: 현재 `BannerItem`은 `{ id, text, timerSeconds?, isTicker? }` 수준이므로, 그대로 사용 가능.
  - 타이머(초)는 나중에 “N초 후 자동 숨김” 등으로 확장 가능.

---

### 2.4 에셋 기능 — 구현 방향

- **목표**: 선택한 에셋(로고, 오버레이, 비디오 클립 등)을 **송출 화면 위**에 겹쳐서 표시.
- **필요 사항**
  1. **상태 끌어올리기**
     - `StudioAssetPanel`의 “선택된 에셋” (`AssetItem | null`)를 StudioMain(또는 컨텍스트)으로 올리기.
     - `onSelectAsset`를 사이드바/패널에 연결.
  2. **PreviewArea에 전달**
     - `activeAsset: AssetItem | null` (및 필요 시 위치/크기) 전달.
  3. **Konva에 에셋 노드 추가**
     - **Image**: `fileUrl`이 있으면 `new window.Image()` 로 로드 후 Konva `Image`에 넣기.
     - URL이 없는 목업 오버레이(예: “Welcome”, “Q&A”)는 **Rect + Text** 조합으로 고정 디자인으로 그리거나, 나중에 백엔드에서 URL 내려주면 Image로 전환.
     - 위치: 예) 중앙 상단, 또는 고정 좌표.
     - zOrder: 소스들보다 **위**에 그리기 (Layer에서 나중에 그리는 노드가 위에 보이면 됨).
- **에셋 타입별**
  - **로고/이미지**: Konva `Image`.
  - **비디오 클립**: HTMLVideoElement를 Konva Image에 넣고, VideoSourceNode와 유사하게 매 프레임 그리기 (구현 복잡도는 높음).
  - 첫 단계에서는 **이미지/오버레이(Rect+Text)**만 해도 됨.

---

### 2.5 스타일 기능 — 구현 방향

- **목표**: “화면의 색상 테마”, “출력 영상의 모양”(예: 원형/사각) 변경.
- **필요 사항**
  1. **상태 연결**
     - `StudioStylePanel`의 `StudioStyleState`(brandColor, theme, showDisplayNames, showHeadlines, font)를 이미 `onStyleChange`로 올릴 수 있게 되어 있음.
     - StudioMain에서 `styleState`를 받아서 PreviewArea에 `styleState`로 전달하면 됨.
  2. **PreviewArea에서 반영**
     - **테마(출력 영상 모양)**
       - 각 소스 **Group**에 `clipFunc`를 지정해서,
         - `bubble` → 원형 클리핑
         - `classic` / `minimal` / `block` → 사각 또는 둥근 사각 등으로 구분.
       - Konva Group은 `clipFunc: (ctx, shape) => { ctx.arc(...); ctx.clip(); }` 형태로 클리핑 가능.
     - **브랜드 색상**
       - 배너 바, 에셋 배경, 텍스트 색 등에 `styleState.brandColor` 적용.
     - **이름/헤드라인 표시**
       - `showDisplayNames`가 true이면 소스 위/아래에 이름 Text 노드 표시 등 (선택 사항).

---

### 2.6 공통 필요 사항 (구현 순서 제안)

1. **StudioMain (또는 스튜디오 컨텍스트)**
   - `activeBanner`, `activeAsset`, `styleState` state 추가.
   - StudioSidebar에 `onSelectBanner`, `onSelectAsset`, `onStyleChange` 전달하고, 패널에서 호출 시 위 state를 갱신.
2. **StudioSidebar / 각 패널**
   - `StudioBannerPanel`에 `onSelectBanner`, `selectedBanner`(또는 선택 ID) 전달.
   - `StudioAssetPanel`에 `onSelectAsset`, `selectedAsset` 전달.
   - `StudioStylePanel`에 `onStyleChange` 전달 (이미 있음).
   - StudioMain에서 이 콜백들이 위 state를 갱신하도록 연결.
3. **PreviewArea**
   - props 확장: `activeBanner`, `activeAsset`, `styleState` (또는 필요한 필드만).
   - 기존 Layer에서 **소스 Group들 그린 뒤**, 같은 Layer(또는 별도 Layer)에:
     - 배너: 하단 Rect + Text (필요 시 티커 애니메이션).
     - 에셋: Image 또는 Rect+Text.
     - 스타일: 소스 Group에 clipFunc 적용, 배너/에셋 색상 적용.
   - `captureStream`은 기존처럼 Layer의 Canvas를 쓰면 되므로, 배너/에셋/스타일이 모두 녹화·송출에 포함됨.
4. **백엔드**
   - 배너/에셋/스타일을 **저장·불러오기**하려면 백엔드 API가 필요.
   - 현재는 목업/로컬 상태로도 “선택한 것을 송출 화면에 쌓는” 동작까지는 **프론트만으로 구현 가능**.

## 3. 정리

- **카메라/마이크/화면공유**:
  - **getUserMedia**(카메라·마이크), **getDisplayMedia**(화면공유)로 MediaStream 획득 →
  - **Konva Stage(Layer)** 위에 **HTMLVideoElement**를 **Image**로 매 프레임 그리기 →
  - Layer의 **Canvas**를 **captureStream**으로 내보내서 녹화·송출.
- **배너/에셋/스타일**:
  - 현재 송출 화면(Konva) 위에 **추가 노드**만 넣으면 되므로 **구현 가능**.
  - 필요한 작업: (1) 배너/에셋/스타일 상태를 StudioMain으로 끌어올리기, (2) PreviewArea에 전달, (3) Konva Layer에 배너용 Rect/Text, 에셋용 Image(또는 Rect+Text), 스타일용 clip/색상 적용.

이 문서를 기준으로 배너 → 에셋 → 스타일 순으로 단계적으로 붙이면, 첨부하신 사진과 같은 동작을 현재 구조 위에서 구현할 수 있습니다.
