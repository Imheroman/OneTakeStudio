# 스튜디오 오른쪽 사이드바 구현 분석 보고서

> 피그마 초안 기준, **오른쪽 사이드바**만 구현 대상이며, 메인 영역(프리뷰, 레이아웃, 씬, 소스)은 다른 프론트엔드 멤버 담당입니다.

---

## 1. 현재 코드 상태 요약

### 1.1 스튜디오 페이지 구조 (프론트)

| 구분     | 경로/컴포넌트                                     | 상태                                                                             |
| -------- | ------------------------------------------------- | -------------------------------------------------------------------------------- |
| 라우트   | `app/(main)/studio/[id]/page.tsx`                 | ✅ 구현됨. `studioId` 추출 후 `StudioMain` 렌더링                                |
| 메인     | `features/studio/studio-main/StudioMain.tsx`      | ✅ 구현됨. 헤더·프리뷰·레이아웃·씬·소스·컨트롤바·**StudioSidebar** 사용          |
| 사이드바 | `widgets/studio/studio-sidebar/StudioSidebar.tsx` | ⚠️ **뼈대만 존재**. 아이콘 5개 + 하단 원형 2개, **패널/활성화 상태/콘텐츠 없음** |

### 1.2 오른쪽 사이드바 현재 구현 (StudioSidebar.tsx)

- **아이콘 목록**: `MessageSquare`(채팅), `Image`(갤러리), `Palette`(스타일), `List`(리스트), `Users`(사용자) — **피그마와 순서/의미 불일치**
- **동작**: 클릭 시 아무 동작 없음. 열리는 패널 없음.
- **미구현**: 배너, 에셋, 노트, 프라이빗 채팅, 녹화 아이콘 및 해당 패널 전체.

### 1.3 백엔드 API 현황 (스튜디오·사이드바 관련)

| API                                                                                                    | 서비스 | 상태          | 비고                                                              |
| ------------------------------------------------------------------------------------------------------ | ------ | ------------- | ----------------------------------------------------------------- |
| `GET/POST /api/studios`, `GET/PATCH/DELETE /api/studios/{id}`                                          | Core   | ✅ 구현       | 스튜디오 CRUD                                                     |
| `GET/POST /api/studios/{id}/members`, `POST .../invite`, `POST .../kick`, `PATCH .../{memberId}`       | Core   | ✅ 구현       | 멤버 목록·초대(이메일 1명)·강퇴·역할 변경                         |
| `GET/POST/PUT/DELETE /api/studios/{id}/scenes`                                                         | Core   | ✅ 구현       | 씬 CRUD                                                           |
| `GET /api/favorites`, `POST /api/favorites`, `DELETE /api/favorites/{id}`, `GET /api/favorites/search` | Core   | ❌ **미구현** | 프론트는 MSW로만 사용 중. 스튜디오 초대 시 "친구 목록"용으로 필요 |
| 채팅(플랫폼 댓글 통합)                                                                                 | -      | ❌ 미구현     | 유튜브/트위치/치지직 댓글 수집·전달 API 없음                      |
| 배너 CRUD                                                                                              | -      | ❌ 미구현     | 배너 문구·타이머·표시 여부 등 API 없음                            |
| 에셋(로고/오버레이/비디오클립) CRUD                                                                    | -      | ❌ 미구현     | 에셋 목록·업로드·삭제 등 API 없음                                 |
| 스타일(테마/색/폰트) 설정                                                                              | -      | ❌ 미구현     | 스튜디오/배너 스타일 저장 API 없음                                |
| 노트(방송 노트)                                                                                        | -      | ❌ 미구현     | 저장·조회 API 없음                                                |
| 프라이빗 채팅                                                                                          | -      | ❌ 미구현     | 멤버 전용 채팅 메시지 API/WebSocket 없음                          |
| 녹화 시작/중지/목록                                                                                    | Media  | ✅ 구현       | `/api/recordings/**` (Gateway 경유)                               |

## 2. 기능별 상세 분석 (수정/신규 구분)

### 2.1 기본 동작: 입장 시 사이드바 비활성 (1번 사진)

| 구분       | 내용                                                                                                                                                                                                                |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **프론트** | **수정** — 현재는 아이콘만 있고 "활성화되지 않은 상태"가 명확하지 않음. 패널이 없고 선택된 아이콘도 없으므로, "초기에는 어떤 탭도 선택되지 않음"을 상태로 두고, 패널은 열리지 않도록만 하면 됨. (추가 UI 수정 최소) |
| **백엔드** | 불필요                                                                                                                                                                                                              |

## 4. 구현 순서 제안 및 필요 사항

### 4.1 백엔드 의존이 적은 것 (우선 구현 가능)

1. **사이드바 공통 구조** — 탭 아이콘 순서/종류 피그마에 맞게 수정, 탭 클릭 시 해당 패널만 열리도록 상태 처리, 초기에는 비활성(패널 닫힘).
2. **노트** — 백엔드에 노트 API만 추가되면 바로 연동 가능. 없으면 로컬 스토리지 등으로 임시 구현 가능.
3. **스타일** — 저장이 없으면 프론트만 상태로 두고 배너/에셋 UI에 반영.

### 4.2 백엔드 스펙이 필요한 것

1. **채팅** — 댓글/채팅 메시지 스키마, 수집·전달 방식(폴링/WebSocket/SSE).
2. **배너** — 배너 스키마, CRUD 경로.
3. **에셋** — 에셋 타입·스키마, 파일 업로드·URL·삭제 API.
4. **사용자 초대** — 친구(즐겨찾기) API 유무, 멀티 초대(body 또는 다중 호출) 방식.
5. **프라이빗 채팅** — 멤버 전용 메시지 스키마·API·실시간 방식.
6. **녹화** — 제목 등 메타데이터 수정 API 유무.

## 5. 백엔드와 합의가 필요한 사항 (요약)

- **채팅**: 플랫폼 댓글 수집·전달 방식 및 스키마.
- **배너/에셋/노트/스타일**: 각각 CRUD 또는 저장 API 및 스키마.
- **친구(즐겨찾기)**: `/api/favorites` 등 API 제공 여부 및 스펙.
- **멤버 초대**: 다중 사용자 초대(한 번에 여러 명) 지원 여부 및 요청 body 형식.
- **프라이빗 채팅**: 스튜디오 멤버 전용 채팅 API·실시간 방식.
- **녹화**: 녹화 메타데이터(제목 등) 수정 API 유무 및 경로.

위 항목은 별도 문서 **「백엔드*스튜디오*사이드바\_문의사항.md」**에 정리하여 백엔드에 전달할 예정입니다.

## 6. 구현을 위해 필요한 것 정리

### 6.1 백엔드에서 필요한 것

| 기능          | 필요 API/정보                                                                       | 우선순위                  |

| 노트          | `GET/PUT /api/studios/{studioId}/note` 및 body 스키마                               | 중간                      |
| 스타일        | 스튜디오별 스타일 저장 API(선택), 스키마                                            | 낮음(로컬만 써도 됨)      |
| 멤버 목록     | `GET /api/studios/{studioId}/members` 동작 여부·응답 스키마 확인                    | 높음                      |
| 사용자 초대   | 즐겨찾기(친구) API 유무, 다중 초대 body 형식                                        | 높음                      |
| 프라이빗 채팅 | 멤버 전용 채팅 API 또는 WebSocket 및 메시지 스키마                                  | 중간                      |
| 녹화          | Gateway 경로·X-User-Id 확인, 제목 수정 API 유무, Recording 응답 스키마              | 높음                      |

### 6.2 프론트엔드에서 필요한 것

| 구분   | 내용                                                                                                                             |
| ------ | -------------------------------------------------------------------------------------------------------------------------------- |
| 공통   | `studioId`를 사이드바·패널에 전달(StudioMain에서 props/context로 전달). 기존 ApiResponse·인증 방식 준수.                         |
| 스키마 | 백엔드 응답이 정해지면 entities 또는 features에 zod 스키마 추가(채팅, 배너, 에셋, 노트, 스타일, 녹화 등).                        |

## 7. 앞으로 구현을 위해 어떻게 해야 하는지

1. **백엔드 답변 수령**
   - **백엔드*스튜디오*사이드바\_문의사항.md**를 백엔드에 전달하고, 각 항목에 대한 답변(API 유무, 엔드포인트, 스키마, 정책)을 받습니다.
   - 백엔드에서 사용 중인 공통 규칙(날짜 형식, id 타입, ApiResponse 형식 등)이 있으면 함께 공유받습니다.

2. **스펙 확정 후 프론트 작업 순서 제안**
   - **1단계**: 사이드바 공통 구조(탭 순서·활성 탭·패널 열기/닫기, 초기 비활성).
   - **2단계**: 백엔드 API가 이미 있는 기능부터 연동 — 멤버 목록, 녹화(시작/중지/목록, 제목 수정 가능 시).
   - **3단계**: 백엔드 스펙이 확정된 기능 — 배너, 에셋, 노트, 사용자 초대(친구 API·다중 초대 형식 반영).
   - **4단계**: 채팅·프라이빗 채팅 — 실시간 수단(WebSocket 등) 스펙 확정 후 연동.
   - **5단계**: 스타일 — 저장 API가 있으면 연동, 없으면 로컬 상태만.

3. **메인 영역(프리뷰) 담당자와의 협의**
   - "선택된 배너", "선택된 에셋", "스타일 설정" 등을 **어떤 형태로 전달할지**(context, store, callback)를 정해 두면, 송출 화면에 반영하는 쪽과 역할이 나뉩니다.

4. **코드 수정 시**
   - 백엔드 답변에 따라 **entities**에 스키마 추가, **shared/api** 또는 기능별 API 함수 추가, **widgets/studio** 아래에 각 패널 컴포넌트를 추가합니다.
   - 기존 스튜디오·멤버 API와 동일한 인증·에러 처리 방식을 따릅니다.
