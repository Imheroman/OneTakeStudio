# 배너 깜빡임 현상: 원인 분석 및 해결 방안

## 1. 깜빡임 현상 요약

| 배너 유형            | 깜빡임 시점                   |
| -------------------- | ----------------------------- |
| **타이머 없는 배너** | 재생 시작 시 1회, 종료 시 1회 |
| **타이머 있는 배너** | 1초마다 반복                  |

---

## 2. 원인 분석: "누가 → 어떤 행동 → 어떤 일 → 깜빡임"

### 2-1. 타이머 있는 배너: 1초마다 깜빡임

**발생 흐름:**

1. **누가**: `StudioMain` 컴포넌트의 `useEffect` (배너 타이머)
2. **어떤 행동**: `setInterval(1000)`이 1초마다 `setBannerRemainingSeconds(remaining)` 호출
3. **어떤 일**:
   - `StudioMain` state 변경 → **StudioMain 전체 리렌더**
   - `useSourceStreams` 훅이 매 렌더마다 `streamIds = Array.from(streamsMap.keys())`로 **새 배열 생성**
   - `PreviewArea`에 전달되는 `availableStreamIds` prop이 **매번 새 참조**로 바뀜
   - `React.memo(PreviewArea)`가 prop 변경으로 판단 → **PreviewArea 리렌더**
   - `react-konva`가 Konva 트리 동기화 → **캔버스 전체 clear + redraw**
4. **결과**: 1초마다 캔버스가 한 프레임 비었다가 다시 그려지며 **깜빡임** 발생

**핵심**: 오른쪽 네브바의 "남은 시간" 숫자만 바뀌는 것이 아니라, 그 state 변경이 부모(`StudioMain`) 리렌더를 유발하고, 이때 `availableStreamIds`가 매번 새 배열로 만들어져서 **프리뷰 캔버스까지 불필요하게 리렌더**되는 것이 원인입니다.  
타이머 숫자만 바꿔서 전체 페이지가 새로 불러와지는 것은 아니지만, React 리렌더 체인 때문에 프리뷰 영역까지 영향을 받습니다.

---

### 2-2. 타이머 없는 배너: 재생/종료 시 깜빡임

**발생 흐름:**

1. **누가**: 사용자가 배너 패널에서 재생/중단 버튼 클릭
2. **어떤 행동**: `handlePlay` / `handleStop` → `onSelectBanner(banner)` → `setActiveBanner(banner | null)`
3. **어떤 일**:
   - `StudioMain`의 `activeBanner` state 변경 → **StudioMain 리렌더**
   - `PreviewArea`에 전달되는 `activeBanner` prop 변경 (null ↔ 배너 객체)
   - `PreviewArea` 리렌더 → `BannerOverlayNode`가 **마운트/언마운트**
   - `react-konva`가 Stage/Layer 트리 갱신 → **캔버스 전체 redraw**
4. **결과**: 배너가 나타나거나 사라질 때 한 번씩 **깜빡임** 발생

**핵심**: 배너 표시 여부가 바뀔 때마다 `activeBanner` prop이 바뀌므로, 이 시점의 리렌더는 **필수**입니다. 다만 Konva가 전체를 다시 그리면서 순간적으로 화면이 비는 구간이 생겨 깜빡임처럼 보입니다.

---

## 3. 해결 방안

### 3-1. 타이머 배너 1초마다 깜빡임 → **해결 가능**

**원인**: `bannerRemainingSeconds` 변경 시 `StudioMain` 리렌더 → `availableStreamIds`가 매번 새 배열 → `PreviewArea` 불필요 리렌더

**해결**: `useSourceStreams`에서 `streamIds`를 `useMemo`로 고정

```ts
// useSourceStreams.ts
const streamIds = useMemo(() => Array.from(streamsMap.keys()), [streamsMap]);
```

`streamsMap`이 바뀌지 않으면 `streamIds` 참조도 유지되어, `bannerRemainingSeconds`만 바뀌는 경우 `PreviewArea`는 리렌더되지 않습니다.

---

### 3-2. 타이머 없는 배너 재생/종료 시 깜빡임 → **완전 제거는 어렵지만 완화 가능**

**원인**: `activeBanner` 변경 시 `BannerOverlayNode` 마운트/언마운트 → Konva 전체 redraw

**가능한 완화 방법**:

1. **배너를 항상 DOM에 두고 opacity로만 제어**

   - `activeBanner`가 null이어도 `BannerOverlayNode`는 렌더하되 `opacity={0}` 또는 `visible={false}`로 숨김
   - 장점: 마운트/언마운트가 없어서 Konva 트리 변경이 적음
   - 단점: 구조 변경이 필요하고, 비활성 시에도 노드가 남아 있음

2. **Konva `batchDraw` 대신 `draw` 최소화**

   - 이미 `scheduleBatchDraw`로 프레임당 1회만 그리도록 되어 있음
   - 추가로 `layerRef.current?.batchDraw()` 호출 타이밍을 줄이는 방향 검토

3. **react-konva의 `useStrictMode` 비활성화**
   - Strict Mode에서의 이중 마운트가 있다면 비활성화로 리마운트 감소 가능 (다만 이 프로젝트에서 사용 여부 확인 필요)

**한계**: 배너를 실제로 추가/제거하는 순간에는 Konva 트리 변경이 필요하므로, 완전한 제거는 어렵습니다. 위 방법으로 **빈도와 강도**를 줄이는 정도가 현실적입니다.
