# 스킬 문서 최신화 작업

## 작업 일시
2026-01-29

## 작업 개요
프로젝트 구조 탐색 후, `.claude/commands/` 하위의 스킬 문서들이 실제 프로젝트 구조와 일치하지 않는 부분을 발견하여 최신화 작업 수행

---

## 발견된 차이점 및 수정 내용

### 1. 데이터베이스 변경 (PostgreSQL → MySQL 통합)

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| Media Service DB | PostgreSQL 15 | **MySQL 8.0 (media_db)** |
| 설명 | Core-MySQL, Media-PostgreSQL 분리 | **단일 MySQL 인프라로 통합** |

**관련 커밋**: `04684d0 refactor(infra): PostgreSQL -> MySQL 통합 마이그레이션`

---

### 2. 서비스 포트 변경

| 서비스 | 변경 전 | 변경 후 |
|--------|---------|---------|
| API Gateway | 8000 또는 8080 | **60000** |
| Media Service | 8081 | **8082** |
| Core Service | 랜덤 (port: 0) | **8080** |

---

### 3. 인프라 구성 변경

#### docker-compose.yml 변경
| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 구성 | MySQL, PostgreSQL, RabbitMQ, Redis, LiveKit, MinIO, Nginx | **MySQL(통합), Redis, LiveKit** |
| 메시지 브로커 | RabbitMQ | **Redis Streams** |
| 스토리지 | MinIO | 제거 (AWS S3 사용) |

---

### 4. 모듈 구조 변경

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| Gateway 모듈명 | `gateway-service` | **`api-gateway`** |
| Config Server | 존재 (`config-service`) | **제거됨** |
| Spring Cloud 버전 | 2024.0.0 | **2025.0.0** |

---

### 5. common 모듈 구조 변경

**변경 전 (문서)**:
```
common/
├── JwtUtil.java
├── TokenBlacklistService.java
├── JwtAuthenticationFilter.java
└── ApiResponse.java
```

**변경 후 (실제)**:
```
common/
├── dto/ApiResponse.java
└── jwt/JwtUtil.java

# TokenBlacklistService, JwtAuthenticationFilter는 core-service/security/에 위치
```

---

### 6. Studio 도메인 구현 상태

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 구현 상태 | "entity, repository만 (CRUD는 팀원 담당)" | **전체 구현 완료** |
| 포함 항목 | entity, repository | **controller, service, dto, entity, exception, repository** |

## 현재 프로젝트 구조 요약

### 서비스 구성
```
┌─────────────────────────────────────────────────────────────┐
│                   Spring Cloud Gateway                       │
│                    (API Gateway / Port: 60000)               │
└─────────────────────────────────────────────────────────────┘
                            │
                            ├─── Eureka Server (Port: 8761)
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼
┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  Core Service    │ │  Media Service   │ │   AI Service     │
│  (Port: 8080)    │ │  (Port: 8082)    │ │  (Port: 8000)    │
│                  │ │                  │ │                  │
│  MySQL (core_db) │ │  MySQL (media_db)│ │  Python/FastAPI  │
└──────────────────┘ └──────────────────┘ └──────────────────┘
```

### 인프라
- **MySQL 8.0**: core_db (Core Service), media_db (Media Service)
- **Redis 7**: 캐시 + Streams 메시지 브로커
- **LiveKit**: WebRTC SFU 서버

### 실행 순서
1. `docker-compose up -d` → MySQL, Redis, LiveKit
2. Eureka Server (port 8761)
3. Core Service (port 8080)
4. Media Service (port 8082)
5. API Gateway (port 60000)
