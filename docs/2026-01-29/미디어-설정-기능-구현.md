# 미디어 설정 기능 구현

> 작성일: 2026-01-29
> 기능명세서: S04 (장치 자동 초기화 및 감지), S10 (실시간 하드웨어 제어)

## 개요

Media Service에 사용자 미디어 설정(영구 저장)과 세션 미디어 상태(실시간 관리) 기능을 추가했습니다.

---

## 구현된 기능

### 1. 사용자 미디어 설정 (UserMediaSettings)
- 기본 카메라/마이크/스피커 디바이스 저장
- 비디오/오디오 품질 설정 (LOW, MEDIUM, HIGH, ULTRA / STUDIO)
- 노이즈 캔슬링, 에코 캔슬링 설정
- 기본 볼륨 레벨 (0-100)

### 2. 세션 미디어 상태 (SessionMediaState)
- 실시간 비디오/오디오 On/Off 토글
- 뮤트 상태 토글
- 현재 사용 중인 디바이스 추적
- 볼륨 레벨 조절

### 3. Stream 연동
- 스트림 참가 시 미디어 상태 자동 초기화
- 스트림 퇴장 시 미디어 상태 자동 종료

## Stream 연동 흐름

```
[프론트엔드]
    │
    ▼
POST /api/v1/media/stream/join
    │
    ├─→ StreamSession 생성 (CONNECTING)
    ├─→ LiveKit 토큰 발급
    └─→ SessionMediaState 자동 초기화 (사용자 기본 설정 적용)

    │
    ▼
POST /api/v1/media/stream/{studioId}/leave
    │
    ├─→ StreamSession 종료 (DISCONNECTED)
    ├─→ LiveKit 참가자 제거
    └─→ SessionMediaState 자동 종료
```

## API 테스트 결과

### 미디어 설정 API
```bash
# 1. 미디어 설정 조회 (신규 사용자 - 기본값 자동 생성)
curl -X GET http://localhost:8082/api/v1/media/settings -H "X-User-Id: 1"
# → videoQuality: HIGH, audioQuality: HIGH, defaultVolumeLevel: 80

# 2. 미디어 설정 저장
curl -X PUT http://localhost:8082/api/v1/media/settings \
  -H "X-User-Id: 1" -H "Content-Type: application/json" \
  -d '{"videoQuality": "ULTRA", "noiseCancellationEnabled": true, "defaultVolumeLevel": 90}'
```

### 세션 상태 API
```bash
# 3. 세션 상태 초기화
curl -X POST "http://localhost:8082/api/v1/media/settings/session/1/init?streamSessionId=100" \
  -H "X-User-Id: 1"

# 4. 비디오 토글
curl -X POST http://localhost:8082/api/v1/media/settings/session/1/video/toggle \
  -H "X-User-Id: 1"
# → videoEnabled: true → false

# 5. 오디오 토글
curl -X POST http://localhost:8082/api/v1/media/settings/session/1/audio/toggle \
  -H "X-User-Id: 1"
# → audioEnabled: true → false

# 6. 뮤트 토글
curl -X POST http://localhost:8082/api/v1/media/settings/session/1/mute/toggle \
  -H "X-User-Id: 1"
# → isMuted: false → true
```

### Stream 연동 테스트
```bash
# 7. 스트림 참가 (미디어 상태 자동 생성)
curl -X POST http://localhost:8082/api/v1/media/stream/join \
  -H "X-User-Id: 99" -H "Content-Type: application/json" \
  -d '{"studioId": 99, "participantName": "TestUser99"}'

# 8. 미디어 상태 자동 생성 확인
curl -X GET http://localhost:8082/api/v1/media/settings/session/99 -H "X-User-Id: 99"
# → streamSessionId 연결됨, isActive: true

# 9. 스트림 퇴장 (미디어 상태 자동 종료)
curl -X POST http://localhost:8082/api/v1/media/stream/99/leave -H "X-User-Id: 99"

# 10. 미디어 상태 자동 종료 확인
curl -X GET http://localhost:8082/api/v1/media/settings/session/99 -H "X-User-Id: 99"
# → 404 MS002 "미디어 상태를 찾을 수 없습니다"
```

## 주요 설계 결정

### 1. 설정 자동 생성
- 신규 사용자가 설정 조회 시 기본값으로 자동 생성
- 별도 초기화 API 불필요

### 2. Stream 연동
- 프론트엔드 API 호출 횟수 최소화
- 스트림 세션과 미디어 상태의 라이프사이클 동기화

### 3. 토글 API 분리
- video/audio/mute 각각 독립 API
- 프론트엔드에서 단일 버튼 클릭으로 상태 변경 가능
