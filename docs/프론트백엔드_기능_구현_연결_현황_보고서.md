| 회원가입 (이메일 인증)     | ✅ `SignupForm`             | ✅ `send-verification`, `verify-email`, `register` | ✅   |

**비밀번호 재설정 미연결 사유**

- 프론트: `apiClient.post("/api/auth/forgot-password", ...)` 호출이 주석 처리됨
- 백엔드: `POST /api/auth/password-reset`, `POST /api/auth/password-reset/confirm` 구현됨

**연결 시 필요 작업**

- `frontend/src/app/(auth)/forgot-password/page.tsx`에서 주석 해제 후 실제 API 호출로 교체
- 비밀번호 재설정 확인 페이지(`/reset-password?token=...`)가 있다면 해당 API 연동

### 2.4 라이브러리 스토리지

| 항목                                    | 프론트               | 백엔드                        | 연결 |
| --------------------------------------- | -------------------- | ----------------------------- | ---- |
| 스토리지 용량 조회                      | ✅ `getStorage`      | ✅ `GET /api/library/storage` | ✅   |
| 스토리지 파일 목록                      | ✅ `getStorageFiles` | ⚠️ 경로 불일치                | ❌   |
| 영상 개수/제한 (videoCount, videoLimit) | ✅ UI 표시           | ❌ 미구현                     | ❌   |
| 삭제까지 남은 일자 (daysUntilDeletion)  | ✅ UI 표시           | ❌ 미구현                     | ❌   |

**문제점**

1. **경로 불일치**

   - 프론트: `GET /api/library/storage/files`
   - 백엔드: `GET /api/storage/files` (StorageController)
   - LibraryController에는 `/storage/files` 엔드포인트 없음

2. **응답 형식**

   - StorageController는 `StorageFilesResponse`를 그대로 반환 (ApiResponse 래퍼 없음)
   - 프론트는 `{ success, data: { files } }` 형식 기대

3. **필드 부족**
   - 백엔드 `StorageResponse`: `videoCount`, `videoLimit` 없음
   - 백엔드 `StorageFileItem`: `daysUntilDeletion` 없음

**연결 시 필요 작업**

- **옵션 A**: LibraryController에 `GET /api/library/storage/files` 추가, ApiResponse로 감싸서 반환
- **옵션 B**: 프론트에서 `/api/storage/files` 호출로 변경, 응답 스키마를 StorageController 형식에 맞게 수정
- 백엔드 `StorageResponse`에 `videoCount`, `videoLimit` 추가 (영상 50개 제한 정책 반영)
- 백엔드 `StorageFileItem`에 `daysUntilDeletion` 추가 또는 프론트에서 `createdAt` 기반 계산

- `GET /api/v1/shorts/status`: 생성 진행 상태/완료 개수 반환

## 4. 우선순위별 조치 사항

### 높음 (실사용에 직접 영향)

1. **저장 공간 파일 목록 연결**

   - LibraryController에 `GET /api/library/storage/files` 추가
   - 또는 프론트를 `/api/storage/files`로 변경 후 응답 스키마 수정
   - `videoCount`, `videoLimit`, `daysUntilDeletion` 필드 정책 확정 후 백엔드/프론트 반영

2. **동영상 업로드 API 구현**
   - Media Service에 업로드 엔드포인트 추가
   - Core Service Recording 생성 플로우와 연동

### 중간 (기능 보완)

3. **비밀번호 재설정 연동**

   - forgot-password 페이지에서 API 호출 활성화
   - reset-password 페이지 및 토큰 검증 플로우 점검

4. **쇼츠 생성 API 구현**
   - AI/미디어 서비스에 쇼츠 생성·상태 조회 API 추가
   - API Gateway 라우팅 설정

### 낮음 (개선 사항)

5. **저장 공간 파일 삭제**
   - 백엔드에 삭제 API 추가
   - 프론트에서 삭제 후 목록 갱신 연동

## 5. MSW 모킹 현황

- `.env.local`: `NEXT_PUBLIC_API_MOCKING` 주석 처리 → **MSW 비활성화**
- MSW 활성화 시: 로그인, 회원가입, 라이브러리, 스토리지, 쇼츠, 채널, 즐겨찾기, 스튜디오 등 대부분 모킹됨
- 실제 백엔드 연동 테스트 시에는 MSW 비활성화 상태로 확인 필요

---

## 6. 결론

- **인증, 워크스페이스, 라이브러리(녹화/다운로드), 채널, 팀 관리, 받은 초대, 알림, 마이페이지**는 프론트·백엔드 모두 구현되어 있고 연결되어 있음.
- **저장 공간 파일 목록**은 경로·형식·필드 불일치로 미연결.
- **쇼츠 생성, 동영상 업로드**는 백엔드 API가 없어 미연결.
- **비밀번호 재설정**은 백엔드 구현 완료, 프론트 연동만 보완하면 됨.

위 우선순위에 따라 저장 공간·업로드·쇼츠·비밀번호 재설정 순으로 연동 및 구현을 진행하면 된다.
