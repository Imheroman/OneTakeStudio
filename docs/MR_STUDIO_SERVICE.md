# MR: Studio Service 구현

## 한 줄 요약
> 방송 스튜디오를 생성하고, 멤버를 초대하고, 화면 구성(씬)을 관리하는 기능을 구현했습니다.

---

## 무엇을 만들었나요?

### 1. 스튜디오 관리
- 새 스튜디오 만들기
- 스튜디오 이름/썸네일 수정
- 스튜디오 삭제

### 2. 멤버 관리
- 이메일로 팀원 초대
- 역할 부여 (매니저/게스트)
- 멤버 강퇴

### 3. 씬(화면 구성) 관리
- 방송 화면 레이아웃 추가
- 씬 순서 변경
- 씬 삭제

---

## 구현한 API (13개)

| 분류 | API 개수 |
|------|----------|
| 스튜디오 | 5개 (생성, 목록, 상세, 수정, 삭제) |
| 멤버 | 4개 (목록, 초대, 역할변경, 강퇴) |
| 씬 | 4개 (목록, 생성, 수정, 삭제) |

---

## 추가된 파일 (35개)

```
core-service/src/main/java/com/onetake/core/studio/
├── controller/   (3개) - API 처리
├── service/      (3개) - 비즈니스 로직
├── entity/       (7개) - DB 테이블 + Enum
├── dto/          (12개) - 데이터 형식
├── repository/   (4개) - DB 접근
└── exception/    (6개) - 에러 처리
```

## 테스트 방법

1. 서버 실행: `cd core-service && mvn spring-boot:run`
2. 브라우저에서 `api-test-studio.html` 열기
3. 로그인 후 각 API 테스트
