# 스튜디오 오른쪽 사이드바 구현 — 백엔드 문의 사항

프론트엔드에서 **스튜디오 페이지 오른쪽 사이드바** 기능(채팅, 배너, 에셋, 스타일, 노트, 사용자 관리, 프라이빗 채팅, 녹화)을 구현하려고 합니다.  
아래 항목에 대해 **백엔드에서 제공 가능한 API·스키마·정책**을 알려주시면, 그에 맞춰 프론트 구현을 진행하겠습니다.

---

## 1. 현재 상황

- **스튜디오 페이지**: `GET /api/studios/{studioId}` 로 상세 조회 후, 메인 영역(프리뷰·레이아웃·씬·소스) + **오른쪽 사이드바**를 렌더링합니다.
- **오른쪽 사이드바**: 현재는 아이콘 5개만 있고, 클릭 시 열리는 패널이나 API 연동은 없습니다.
- **구현 예정**: 피그마 기준으로 (1) 채팅 (2) 배너 (3) 에셋 (4) 스타일 (5) 노트 (6) 사용자 관리 (7) 프라이빗 채팅 (8) 녹화 — 각 탭별 패널을 만들고, 필요한 API와 연동하려고 합니다.

---

## 2. 프론트엔드에서 수정·생성 예정인 부분 (요약)

| 구분     | 내용                                                                                                                                   |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **수정** | `StudioSidebar.tsx` — 탭 아이콘 순서/종류를 피그마에 맞게 변경, 탭 클릭 시 해당 패널만 열리도록 상태 처리, 초기에는 비활성(패널 닫힘). |
| **신규** | 채팅·배너·에셋·스타일·노트·멤버·프라이빗채팅·녹화용 **패널 컴포넌트** 및 각 기능별 API 클라이언트·스키마(zod 등) 추가.                 |

수정·생성 시 **백엔드 응답 형식(ApiResponse 래퍼, 필드명, 타입)** 및 **인증 방식(JWT, X-User-Id 등)**은 기존 스튜디오/멤버 API와 동일하게 맞추겠습니다.  
백엔드에서 사용 중인 공통 규칙(예: 날짜 ISO8601, id 타입 number vs string)이 있으면 알려주시면 그에 맞춥니다.

---

## 3. 기능별로 백엔드에 요청하는 사항

### 3.1 채팅 (플랫폼 댓글 + 방송 진행자 채팅)

- **기능 요약**: 유튜브/트위치/치지직 등 동시 송출 플랫폼에서 들어온 댓글을 스튜디오 화면에 표시하고, 방송 진행자가 보내는 채팅도 표시하려고 합니다.
- **질문**
  1. 플랫폼 댓글을 수집·저장·전달하는 API 또는 실시간 수단(WebSocket/SSE 등)이 있나요?
  2. 있다면 **엔드포인트·메서드·요청/응답 스키마**(메시지 ID, 플랫폼, 작성자, 내용, 시각 등)를 공유해 주실 수 있을까요?
  3. 방송 진행자 채팅 전송용 API가 있나요? 있다면 **요청 body 스키마**(studioId, userId, content 등)를 알려주세요.
  4. 없다면, 채팅/댓글 기능은 **프론트에서 목업(MSW 등)으로만 구현**해도 될지, 아니면 추후 백엔드 구현 예정이 있는지 알려주시면 됩니다.

---

### 3.2 배너

- **기능 요약**: 배너 문구·타이머(표시 시간) 등을 입력해 배너를 추가하고, 목록에서 선택한 배너를 송출 화면 하단에 표시하려고 합니다. 삭제도 필요합니다.
- **요청**
  1. 스튜디오별 배너 **CRUD API**를 제공할 예정인가요?
  2. 제공한다면 아래 정보를 알려주세요.
     - **엔드포인트**: 예) `GET/POST/DELETE /api/studios/{studioId}/banners` 등
     - **응답/요청 스키마**:
       - 배너 한 건: `id`, `studioId`, `text`, `timerSeconds`(optional), `isTicker`(optional), `sortOrder`, `createdAt` 등 필요한 필드
     - **생성 시 필수 필드**: 문구(text)만인지, 타이머/티커 여부 등 추가 필드가 있는지

---

### 3.3 에셋 (로고, 오버레이, 비디오 클립)

- **기능 요약**: 로고·오버레이·비디오 클립 등을 업로드하고, 목록에서 선택한 에셋을 송출 화면에 표시하려고 합니다. 삭제도 필요합니다.
- **요청**
  1. 스튜디오별 에셋 **저장·목록·삭제 API**를 제공할 예정인가요?
  2. 제공한다면 아래 정보를 알려주세요.
     - **엔드포인트**: 예) `GET/POST/DELETE /api/studios/{studioId}/assets` 등
     - **타입 구분**: 로고/오버레이/비디오클립을 `type` 등으로 구분하는지
     - **업로드 방식**: `multipart/form-data`로 파일 업로드 후 **저장된 파일 URL**(또는 경로)을 응답으로 받는 형태인지
     - **응답 스키마**: `id`, `studioId`, `type`, `name`, `fileUrl`(또는 `filePath`), `createdAt` 등

---

### 3.4 스타일 (테마, 색상, 폰트, 이름/헤드라인 표시)

- **기능 요약**: 배너·에셋 등의 테마·브랜드 색상·폰트, "이름 표시"/"헤드라인 표시" 여부를 설정하려고 합니다.
- **질문**
  1. 스튜디오별(또는 방송별)로 위 설정을 **저장하는 API**가 있나요?
  2. 있다면 **엔드포인트·스키마**(예: theme, brandColor, font, showDisplayNames, showHeadlines 등)를 알려주세요.
  3. 없다면 **프론트만 로컬/세션 상태**로 두고, 저장 없이 동작하게 해도 될지 알려주시면 됩니다.

---

### 3.5 노트 (방송 노트)

- **기능 요약**: 방송 중 참고용으로 텍스트를 적어 두고, 같은 스튜디오에서 불러와 보려고 합니다.
- **요청**
  1. 스튜디오당 **노트 1건** 저장/조회 API를 제공할 예정인가요?
  2. 제공한다면 예시만 알려주시면 됩니다.
     - 예) `GET /api/studios/{studioId}/note` → `{ content: string }`
     - 예) `PUT /api/studios/{studioId}/note` body `{ content: string }`

---

### 3.6 사용자 관리 — 멤버 목록

- **현재**: `GET /api/studios/{studioId}/members` 가 이미 문서/코드에 있습니다.
- **요청**
  1. 실제로 **동작 중인지**와, **응답 스키마**(멤버 id, userId, nickname, email, role, joinedAt 등)를 한 번만 확인하고 싶습니다.
  2. **ApiResponse** 래퍼 사용 여부(`{ success, data }` 등)와 `data` 안에 배열이 오는지 알려주시면, 그에 맞춰 프론트 스키마를 맞추겠습니다.

---

### 3.7 사용자 초대 — 친구(즐겨찾기) 목록 + 다중 초대

- **기능 요약**: "사용자 초대" 시 (1) **친구(즐겨찾기) 목록**에서 체크로 여러 명 한 번에 초대하거나, (2) **이메일**로 1명씩 초대하려고 합니다.
- **현재 백엔드**: `POST /api/studios/{studioId}/members/invite` 는 `InviteMemberRequest { email, role }` 로 **이메일 1명**만 받는 것으로 보입니다.
- **질문**
  1. **즐겨찾기(친구) API**가 있나요?
     - 예) `GET /api/favorites`, `GET /api/favorites/search?q=...`
     - 없다면, 스튜디오 초대 시 "친구 목록에서 선택"은 **제외**하고 **이메일 입력만** 지원해도 될지 알려주세요.
  2. **한 번에 여러 명 초대**를 지원할 예정인가요?
     - 지원한다면 요청 body 예시를 알려주세요. (예: `{ userIds: string[], role: string }` 또는 `{ invites: { email, role }[] }` 등)
     - 지원하지 않으면, 프론트에서 **1명씩 여러 번 호출**하는 방식으로 진행해도 될지 확인 부탁드립니다.

---

### 3.8 프라이빗 채팅 (멤버 전용 채팅)

- **기능 요약**: 스튜디오 멤버(7번에서 등록된 사용자)만 참여하는 채팅창을 만들려고 합니다. 일반 채팅과 유사하게 메시지 목록·입력·전송·이모티콘을 넣을 예정입니다.
- **요청**
  1. 스튜디오별 **멤버 전용 채팅** API 또는 실시간 전달(WebSocket/SSE 등)을 제공할 예정인가요?
  2. 제공한다면 **엔드포인트·메시지 스키마**(id, studioId, userId, content, createdAt 등)·인증 방식을 알려주세요.
  3. 없다면, **프론트에서 목업으로만** 구현해 두고, 추후 백엔드 연동할지 여부만 알려주셔도 됩니다.

---

### 3.9 녹화

- **현재 파악**: Media 서비스에
  - `POST /api/v1/media/record/start` (Gateway: `/api/recordings/start`)
  - `POST /api/v1/media/record/{studioId}/stop` (Gateway: `/api/recordings/{studioId}/stop`)
  - `GET /api/v1/media/record/studio/{studioId}` (Gateway: `/api/recordings/studio/{studioId}`)  
    등이 있는 것으로 보입니다.
- **요청**
  1. 위 경로가 **API Gateway(60000)** 를 통해 **실제로 노출되는지** 확인 부탁드립니다. (예: `POST https://localhost:60000/api/recordings/start`, body `{ studioId }`, 헤더 `X-User-Id` 등)
  2. **녹화 메타데이터(제목 등) 수정** API가 있나요?
     - 예) `PATCH /api/recordings/{recordingId}` body `{ title: string }`
     - 없다면, "제목 수정"은 **프론트에서 비활성화**하거나, 나중에 백엔드 추가 시 연동하겠습니다.
  3. **RecordingResponse** 에 들어가는 필드 중 프론트에서 꼭 쓰고 싶은 것: `recordingId`, `studioId`, `status`, `fileName`, `s3Url`, `durationSeconds`, `startedAt`, `endedAt`, `createdAt`. **제목(title)** 필드가 있으면 알려주세요.

## 4. 그 외 코드 작성 시 백엔드에서 필요한 사항

1. **공통 응답 형식**
   - 스튜디오·멤버 API처럼 `{ success: boolean, message?: string, data: T }` 를 쓰는지,
   - 아니면 `data` 없이 바로 배열/객체를 주는 API가 있는지 알려주시면, 프론트에서 일괄 처리하겠습니다.

2. **인증**
   - 스튜디오 관련 API 호출 시 **Authorization: Bearer {token}** 만 쓰면 되는지,
   - **X-User-Id** 등 추가 헤더가 필수인 API가 있다면 경로와 함께 알려주세요.

3. **에러 응답 형식**
   - 4xx/5xx 시 body 형식(예: `{ code, message }`)이 정해져 있으면 알려주시면, 프론트에서 에러 메시지 표시에 활용하겠습니다.

4. **CORS**
   - 프론트 개발 서버(localhost:3000)에서 API Gateway(localhost:60000)로 요청할 때, **CORS 설정**이 이미 허용되어 있는지 확인 부탁드립니다.

---

## 5. 정리

- **채팅·배너·에셋·노트·스타일·프라이빗채팅**: 제공 예정인 API가 있으면 **엔드포인트·메서드·요청/응답 스키마**를 알려주시면, 그에 맞춰 프론트 사이드바 패널과 스키마를 작성하겠습니다. 없으면 목업 또는 저장 없이 UI만 구현하는 방식으로 진행하겠습니다.
- **사용자 초대**: 친구(즐겨찾기) API 유무, 다중 초대 지원 여부와 body 형식을 알려주시면, 그에 맞춰 초대 모달을 구현하겠습니다.
- **녹화**: Gateway 경로·헤더·Recording 응답 스키마와, 제목 수정 API 유무만 확인해 주시면 됩니다.

백엔드 팀에서 위 항목에 대해 회신해 주시면, 그 내용을 반영한 뒤 구현을 진행하겠습니다.
