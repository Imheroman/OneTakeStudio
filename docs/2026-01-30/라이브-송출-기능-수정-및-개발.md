## 1. Gateway 라우팅 수정

### 문제
- Gateway에서 Media Service로 라우팅 시 경로 불일치로 500 에러 발생
- `/api/v1/media/` 경로로 리라이트했으나 실제 컨트롤러는 `/api/media/` 사용

### 해결
**파일:** `api-gateway/src/main/resources/application.yml`

```yaml
# Media Service - Publish Routes (root: POST /api/publish → /api/media/publish)
- id: media-service-publish-root
  uri: lb://media-service
  predicates:
    - Path=/api/publish
  filters:
    - RewritePath=/api/publish, /api/media/publish

# Media Service - Publish Routes (sub paths)
- id: media-service-publish
  uri: lb://media-service
  predicates:
    - Path=/api/publish/**
  filters:
    - RewritePath=/api/publish/(?<segment>.*), /api/media/publish/${segment}
```

### 해결
**파일:** `core-service/src/main/java/com/onetake/core/config/SecurityConfig.java`

```java
http

```

- Core Service에서 CORS 비활성화
- Gateway에서만 CORS 처리하도록 단일화

## 3. YouTube 송출 화면 수정 (Konva 캔버스 적용)

### 문제
- YouTube 송출 시 레이아웃이 적용된 화면이 아닌 raw 카메라 영상만 출력
- `createLocalTracks({ video: true, audio: true })`로 기본 카메라만 캡처

### 해결
**파일:** `frontend/src/features/studio/studio-main/model/useStudioMain.ts`

Konva 캔버스에서 합성된 스트림을 LiveKit에 퍼블리시하도록 변경:

```typescript
// LiveKit 연결 시 Konva 캔버스 스트림 사용
const getPreviewStreamRef = options?.getPreviewStreamRef;

const handleGoLive = useCallback(async (destinationIds?: number[]) => {
  // ... 연결 설정 ...

  await room.connect(tokenResponse.livekitUrl, tokenResponse.token);

  // Konva 캔버스 스트림에서 비디오 트랙 가져오기 (레이아웃 적용된 화면)
  const canvasStream = getPreviewStreamRef?.current?.();
  if (canvasStream) {
    const videoTracks = canvasStream.getVideoTracks();
    if (videoTracks.length > 0) {
      await room.localParticipant.publishTrack(videoTracks[0], {
        name: "canvas",
        source: Track.Source.Camera,
      });
    }
  } else {
    // 캔버스 스트림이 없으면 기본 카메라 사용 (fallback)
    const videoTrack = await createLocalTracks({ video: true, audio: false });
    for (const track of videoTrack) {
      await room.localParticipant.publishTrack(track);
    }
  }

  // 오디오 트랙 별도 처리 (마이크)
  try {
    const audioTracks = await createLocalTracks({ video: false, audio: true });
    for (const track of audioTracks) {
      await room.localParticipant.publishTrack(track);
    }
  } catch (audioErr) {
    console.warn("Failed to get audio track:", audioErr);
  }
}, [studioId, user?.nickname, selectedDestinationIds, getPreviewStreamRef]);
```

## 4. TypeScript 타입 오류 수정

### 문제
- `livekit-room.ts`에서 `source` 파라미터에 문자열 리터럴 사용으로 타입 오류 발생
- `"camera"`, `"microphone"` → `Track.Source` enum 필요

### 해결
**파일:** `frontend/src/shared/lib/livekit-room.ts`

```typescript
// 수정 전
import { Room } from "livekit-client";
source: "camera"
source: "microphone"

// 수정 후
import { Room, Track } from "livekit-client";
source: Track.Source.Camera
source: Track.Source.Microphone
```

## 5. LiveStreamEventPublisher 생성

### 문제
- `PublishService.java`에서 `LiveStreamEventPublisher` 클래스를 import하지만 해당 클래스 미존재
- 컴파일 오류 발생

### 해결
**파일:** `media-service/src/main/java/com/onetake/media/global/event/LiveStreamEventPublisher.java` (신규 생성)

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class LiveStreamEventPublisher {

    private final StringRedisTemplate redisTemplate;
    private static final String STREAM_KEY = "live-stream-events";

    /**
     * 라이브 시작 이벤트 발행
     */
    public void publishLiveStarted(Long studioId, Long userId, String roomName) {
        Map<String, String> event = new HashMap<>();
        event.put("eventType", "LIVE_STARTED");
        event.put("studioId", String.valueOf(studioId));
        event.put("userId", String.valueOf(userId));
        event.put("roomName", roomName);
        event.put("timestamp", Instant.now().toString());

        try {
            MapRecord<String, String, String> record = StreamRecords
                    .mapBacked(event)
                    .withStreamKey(STREAM_KEY);
            redisTemplate.opsForStream().add(record);
        } catch (Exception e) {
            log.warn("Failed to publish event: {}", e.getMessage());
        }
    }

    /**
     * 라이브 종료 이벤트 발행
     */
    public void publishLiveEnded(Long studioId, Long userId, String roomName) {
        // 동일한 구조로 LIVE_ENDED 이벤트 발행
    }
}
```

## 6. 신규 API 클라이언트 추가

### Stream API
**파일:** `frontend/src/shared/api/studio-stream.ts`

- `joinStream()` - 스트림 참가 (LiveKit 토큰 발급)
- `leaveStream()` - 스트림 퇴장
- `endStream()` - 스트림 종료 (호스트만)
- `getActiveSession()` - 활성 스트림 세션 조회

### Publish API
**파일:** `frontend/src/shared/api/studio-publish.ts`

- `startPublish()` - RTMP 송출 시작
- `stopPublish()` - RTMP 송출 중지
- `getPublishStatus()` - 송출 상태 조회
