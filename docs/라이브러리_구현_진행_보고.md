## 2. 백엔드 관점 정리

### 2.1 이번 작업에서 백엔드가 담당하는 것 (구현 예정/이미 구현)

| 항목                                          | 담당          | 비고                                                                     |
| --------------------------------------------- | ------------- | ------------------------------------------------------------------------ |
| `GET /api/library/videos?type=&studioId=`     | Core-Service  | 통합 비디오 목록, 응답 `{ success, data: { videos, total } }`            |
| `GET /api/library/videos/{videoId}`           | Core-Service  | 비디오 상세, 응답 `{ success, data: VideoDetail }`, data.clips 포함      |
| `POST /api/library/clips`                     | Core-Service  | 클립 생성(트림 저장), Body: recordingId, title, startTimeSec, endTimeSec |
| `GET /api/library/videos/{videoId}/download`  | Core-Service  | 단건 다운로드                                                            |
| `POST /api/v1/media/upload`                   | Media-Service | Form: studioId, title, file                                              |
| API Gateway `/api/library/**`                 | 인프라        | Core-Service로 라우팅                                                    |
| clips 테이블·로컬 업로드(recordings.filePath) | DB/백엔드     | 이미 반영됨(답변 기준)                                                   |

### 2.2 백엔드 변경 사항 (이번 협의에서 확정된 것)

- Library API: Core-Service, 경로 `/api/library/videos` 통합.
- ID: recordingId/clipId UUID 문자열.
- 응답: `{ success: boolean, data: T }` 래퍼.
- 비디오 목록: 쿼리 `studioId` 필수, `type=all|original|shorts` 필터.
- 트림 저장: PUT trim 대신 `POST /api/library/clips`로 클립 생성.
- 업로드: Media-Service `POST /api/v1/media/upload`, FormData.

## 3. 프론트엔드 관점 정리

### 3.1 진행한 작업 (구현 완료)

1. **공통**
   - Video 스키마: `id`를 `number` → `string`(UUID) 변경.
   - Library API 응답: `VideoListApiResponseSchema`, `VideoDetailApiResponseSchema` 등 `{ success, data }` 래퍼 스키마 추가 및 연동.
   - apiClient: 모든 요청에 `X-User-Id` 헤더 추가(useAuthStore `user.userId`).
   - apiClient: `postForm(url, formData)` 추가(업로드용).

2. **라이브러리 목록 페이지**
   - URL 쿼리 `studioId` 사용: `useSearchParams().get("studioId")` → VideoLibrary에 `studioId` 전달.
   - `studioId` 없을 때: "스튜디오를 선택해 주세요" 안내 문구 표시.
   - `GET /api/library/videos?type=&studioId=` 호출, 응답 `data.videos` / `data.total` 사용.
   - "Upload File" 버튼 추가, 클릭 시 업로드 모달 오픈.

3. **업로드**
   - `UploadVideoModal` 위젯 추가: 드래그 앤 드롭, 파일 선택, 제목(선택), `POST /api/v1/media/upload` (FormData: studioId, title, file).
   - 업로드 성공 시 목록 재조회 및 모달 닫기.

4. **라이브러리 상세 페이지**
   - `GET /api/library/videos/{videoId}` 로 상세 조회, 로딩/에러 처리.
   - Back 링크: `/library`로 이동.
   - VideoPlayer: `videoUrl` prop으로 `<video>` 재생.
   - VideoInfoSection: API에서 받은 title, date, description 표시, Download/Save/Generate Shorts 버튼 연동.
   - VideoSidebar: `video.clips`로 원본+쇼츠 목록 표시, 클립 클릭 시 쇼츠 재생 모달 오픈.
   - TrimSection: 시작/종료 구간 슬라이더, Save 시 `POST /api/library/clips` 호출 후 상세 재조회.
   - DownloadVideoModal: 단건 다운로드(인증 헤더 + blob 처리).
   - ShortsPlaybackModal: clip.url 재생, 재생/일시정지 버튼, 닫기.

5. **MSW**
   - `GET /api/library/videos`: 응답을 `{ success, data: { videos, total } }` 형식으로 변경, id 문자열(rec-1, clip-4 등).
   - `GET /api/library/videos/:videoId`: 추가, `{ success, data: VideoDetail }` 형식.
   - `POST /api/library/clips`: 추가, Body 검증 후 `{ success, data }` 반환.
